# Contributing to RM
1. Основная ветка: `main`
2. Разработка: `develop`
3. Фича-ветки: `feat/<scope>`
4. Багфиксы: `fix/<scope>`
5. Все PR проходят ревью и CI-проверку.
6. Перед пушем: `pnpm lint` и `pnpm test`.
7. В PR описывать:
   - Цель
   - Что сделано
   - Критерии приёмки
   - Список изменённых файлов
